◆ How I Cut MTTR by 40% with a 3-Agent AutoGen SystemBlog
EngineeringAgentic AI

Major incidents used to mean the same thing every time: a war-room bridge, a scramble to page the right people, and hours of manual RCA writing after the fact. Here's how a 3-agent system changed that.

The core problem wasn't a lack of process — it was that every major incident consumed the same manual steps: classify severity, spin up a Teams bridge, notify the right stakeholders, and, after resolution, write up a root cause analysis nobody had time to do properly.

I architected a 3-agent system using Microsoft AutoGen to split that work into specialists: an Incident Analysis Agent that classifies severity and drafts the RCA, a Meeting Agent that creates the Teams bridge and notifies stakeholders via Microsoft Graph API, and a Transcript Analysis Agent that extracts action items once the call ends. AutoGen coordinates the handoffs between them.

Underneath, Azure OpenAI's GPT-4o handles the real-time analysis and RCA drafting, exposed through async FastAPI endpoints so multiple incidents can be processed concurrently. The result: MTTR down ~40% across 30+ major incidents a month, and roughly 15 engineer-hours a week no longer spent on manual triage and documentation.

◆ RAG in Production: What Actually Reduces HallucinationsBlog
RAGApplied AI

An LLM confidently inventing a root cause is worse than no answer at all. What actually made retrieval reliable in a live incident-response pipeline.

When I connected GPT-4o to incident RCA generation, the first version leaned entirely on the model's own reasoning — and it occasionally produced plausible-sounding but incorrect root causes. The fix wasn't a bigger model, it was better grounding.

I built a RAG pipeline using Azure OpenAI embeddings with PostgreSQL's pgvector extension, indexing historical incidents and existing runbooks. Before generating a response, the system retrieves the most relevant past incidents and known fixes, and the model is instructed to reason from that retrieved context rather than from memory alone.

The practical takeaway: retrieval quality mattered more than prompt cleverness. Getting the right historical incidents into context — not just semantically similar ones — is what actually cut down on hallucinated resolutions and made the output something engineers could trust during a live incident.

◆ Building a Self-Healing Infra Framework (Auto Heal)Blog
AutomationInfra

Patch validation across hundreds of Linux and Windows servers was eating 10+ hours a week in manual checks. Auto Heal was built to catch and fix the common failures before anyone had to.

Working with Bayer's server estate, patch validation followed a predictable pattern: run pre-checks, apply the patch, run post-checks, and manually fix whatever broke. Most of the breakages were the same handful of failure patterns, repeated across hundreds of machines.

Auto Heal is a self-healing engine, built in Python and PowerShell, that automatically detects and remediates those common infrastructure issues across both Linux and Windows — before they need a human on-call response. It handles health monitoring, compliance validation, and patch verification end-to-end, and exposes its automation through FastAPI so it plugs into broader enterprise tooling.

Net result: manual patch-validation effort dropped by 50%, saving over 10 engineer-hours a week that used to go into repetitive checks instead of actual problem-solving.

◆ Why I Build Personal AI Agents on the SideBlog
PersonalAgentic AI

Client work teaches you to build within constraints. Personal projects teach you what you'd build without them — that's why I keep a few running unattended, end to end.

My day-to-day work is enterprise incident automation — scoped, reviewed, and shipped within a client's constraints. My personal projects are the opposite: small, fast, and built purely to test an idea end-to-end without anyone signing off on the architecture first.

That's produced three unattended pipelines so far: a lead-scoring agent chain that takes a form submission straight to a scored, personalized email with no human step; an autonomous LinkedIn publisher that researches, drafts, edits, and posts technical content on a schedule; and a job-alert pipeline that reads Gmail alerts, scores them against my own profile, and logs only what's actually relevant to Sheets.

None of them are complicated individually — the value is in each one running completely on its own, trigger to output. That's the muscle I'm building: not just using AI agents, but trusting them to run without me in the loop.
